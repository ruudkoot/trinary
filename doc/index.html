Overview

Introduction

The Trinary Operating System is a new operating system designed using the most modern and advanced technologies. The keywords here are stability, simplicity and flexibility. That this goal is achieved at the cost of speed and compatibility is acceptable.

Features

The current version of the Trinary Operating System is in an alpha-state, so most features listed below have not yet been implemented.

  • A modern object-orientated API
  • An advanced high-performance file system
  • A revolutionary device-management system

Kernel

To help us enhancing this OS run the most current version of TriOS on as many computers (with a different hardware configuration) you have access to. If the OS reports "Uncertified Hardware Detected", please make a report with the information given about that hardware component, say if the hardware was correctly detected, and if not, how it should be detected. And finally mail this report to rudykoot@trinary.tk.

  • Trinary Operating System 03.04.12 (Alpha)

    • Started working on SMP support.

  • Trinary Operating System 03.03.02 (Alpha)

    • This version is been ported to GCC. Except for the Run Time Library most of the code is completly rewritten. Although it no longer has ATA support it does have better exception handling, which you hopefully don't see and multitasking, finaly.

    Each of the counter you see is a seperate task. You can notice that by the fact that they are noticably counting out of sync after about a minute. At the moment they are kernel tasks, meaning that they run at privilege level 0. This will change in the next release which will hopefully be out sooner than this version.

    Every 20 seconds or so the screen gets messed up and the Interrupt Counter in the title bar indicates that and IRQ7 has occured. This a problem with the interrupt controller and is called a Spurious IRQ7. A Surious IRQ7 occures when the Interrupt Controller can't handle the IRQs fast enough. The is caused by the Interval Timer which raises IRQ0 close to 100 000 times a second. As a comparrison: under DOS/Windows 3.1/Windows 9x IRQ0 is raised only 18.7 times a second and under Windows NT/2K/XP 1000 times a second. I's better slow it down somewhat in the next release.

  • Trinary Operating System 03.02.07 (Alpha)

    • Interrupt support was added as well as some basic ATA disk support. The title bar contains an interrupt counter. The model ofthe device connected to the master or slave of your primary IDE port is displayed.

  • Trinary Operating System 03.02.01 (Alpha)

    • The previous versions would only run on a Pentium II, Athlon or higher, because of a wrong compiler setting. This version work on a 386 and up. Corrected some errors in the CPU detection code. The light of the floppy drive is now turned off and the A20 is enabled. Some major work was done to the run-time library.

  • Trinary Operating System 03.01.28 (Alpha)

    • This version has paging enabled, and some basic memory management functions. The CPU detection also works correctly now. A small bug in the strcmp function caused it to fail on the Pentium II, Pentium III, Pentium 4 and AMD Athlons.

  • Trinary Operating System 03.01.27 (Alpha)

    • This version fixes the bugs in the previous ones.

  • Trinary Operating System 03.01.23 (Alpha)

    • TriOS can now identify your processor and will display it's name on the screen. Anything form a 486 to a Pentium 4 to a Cyrix MediaGX can be identified. Due to the simplicity of the current bootloader, the kernel isn't loaded completely in memory, and some of the text displayed will be messed up.

  • Trinary Operating System 03.01.20 (Alpha)

    • This is the first public release of the Trinary Operating System. It has a basic bootloader and kernel. Due to a small bug, it will only load under an emulator, an won't load on a real computer.

Development

  • Development Tools for Trinary Operating System 03.03.01

    • This archive contains a minimal DOS port of GCC 2.96 (DJGPP 2.03 with Binutils 2.9), NASM 0.98 and the Bochs emulator (2.0.pre3). If you want to build TriOS under any other OS just use the GCC that comes with your OS and look at the build.bat file to see how to build what files. You'll have to be original with the files that need to be assembled with NASM (extract them from the disk image for example). I'll port them to GAS soon.

  • Development Tools for Trinary Operating System 03.02.01 - 03.02.07

    • This archive only includes NASM and the Visual Studio workspace. You still need Microsoft Visual C++ (see below). TCC is no longer required.

  • Development Tools for Trinary Operating System 03.01.20 - 03.01.28

    • This archive contains some of the tools needed to build the kernel and a disk image. These include NASM, TCC and some scripts. You will also need Microsoft Visual C++ to build most of the C files, however. Alternatively you could try Microsoft Visual C.NET or the Intel Optimizing C++ Compiler.

  • Emulator for Trinary Operating System 03.02.01

    • Bochs was rearchived, because I changed the directory structure of the distribution.

  • Emulator for Trinary Operating System 03.01.20 - 03.01.28

    • The emulator Bochs allows you to run the Trinary Operating System without installing it on your computer. Visit their site at bochs.sourceforge.net.

Supporting Trinary
